# Sector — the food-security landscape

Foundational context so a builder understands the world these [personas](../personas/) live in — **who
the players are** and **why the problems stay unsolved**. This is the "subject-matter expert voicing the
customer" that a nonprofit partner would normally provide in person, drawn from the ESRI **Generate and
Design** event (Chesterbrook, PA; facilitated by TechSoup and TechImpact) and SME input.

> **Status: draft.** Both docs are to be enriched from the sector source report once it's in the drive.

| Doc | What it covers |
|---|---|
| [landscape.md](landscape.md) | Who the players are — food banks, pantries, food cupboards, the Feeding America network line, and who a builder should co-design with |
| [why-this-is-hard.md](why-this-is-hard.md) | The non-obvious constraints that keep these problems unsolved, at the user, provider, and system levels |

The "problem" itself isn't a separate folder — it's the **gap** captured in
[why-this-is-hard.md](why-this-is-hard.md) and in each [design](../designs/)'s how-might-we question. See
[jobs-to-be-done/](../jobs-to-be-done/) for how these constraints map onto buildable work.
